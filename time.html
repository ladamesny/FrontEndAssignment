<time>

<time> is used to tag time elements in a machine readable way so that user agents can use it to add an event to the user's calendar.

Below we're nesting <details> elements to toggle the view of a shoppping list and subsections, which we define with <summary>tags.

  • 13th November: [when using @datetime the content can be anything relevant]

  • 8pm on my birthday: [datetime with time-zone example]
  • 8pm on my birthday: [datetime with time-zone "Z"]
